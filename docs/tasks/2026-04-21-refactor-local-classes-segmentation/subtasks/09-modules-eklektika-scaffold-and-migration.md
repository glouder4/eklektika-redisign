## Цель подзадачи

Подготовить структуру Bitrix-модулей в `local/modules/` с префиксом **`eklektika.`**, перенести туда автозагрузку и `include.php`, запланировать постепенную смену namespace с `OnlineService\...` на `Eklektika\...` без ломки шаблонов (этапы с `class_alias` или параллельные классы-обёртки). Модули вне префикса `eklektika.*` не затрагиваются (см. arch-rules).

## Описание работ

1. Создать минимальный каркас для приоритетных модулей из таблицы в [README](../README.md): например `eklektika.b24.rest`, `eklektika.company`, `eklektika.catalog.pricing` — каждый с `include.php`, `install/index.php` (минимально для локальной установки), `.settings.php` при необходимости.
2. Зарегистрировать автозагрузку через `Loader::registerAutoLoadClasses` в `include.php` модулей.
3. Определить политику имён: либо сохранить `OnlineService\*` внутри модулей до второй волны, либо ввести `Eklektika\*` и алиасы — зафиксировать в документации.
4. В `Контент > Установленные решения` (или через скрипт) выполнить установку модулей на dev-стенде; обновить `init.php` на `CModule::IncludeModule('eklektika.*')` вместо ручных `require` где уместно.

## Технические детали

- Компоненты/модули:
  - новые: `local/modules/eklektika.*/`
  - не трогать: сторонние модули вне префикса `eklektika.*` (перечень запретов — в arch-rules)
- Изменяемые файлы/области:
  - `local/modules/eklektika.*/**`
  - `local/php_interface/init.php` — подключение модулей
  - постепенно — namespace в `local/classes` при переносе файлов
