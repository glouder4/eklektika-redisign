# Интеграция синхронизации компаний с Bitrix24

**Дата:** 20 октября 2025  
**Файлы:** 
- `local/classes/site/Company.php` (метод `sendToBitrix24()`)
- `local/components/online-service/company.profile.edit/ajax.php`

## 🎯 Цель интеграции

Обеспечить автоматическую синхронизацию данных компании с CRM Bitrix24 при обновлении профиля через веб-интерфейс сайта.

## 📋 Описание функциональности

При обновлении профиля компании через компонент `company.profile.edit`, изменения автоматически отправляются в Bitrix24 CRM для синхронизации данных.

## 🔧 Реализация

### Новый метод: `sendToBitrix24()`

```php
/**
 * Отправить обновленные данные компании в Bitrix24
 *
 * @param int $companyId - ID компании в Bitrix (из CODE элемента)
 * @param array $data - данные компании для отправки
 * @param bool $debug - режим отладки
 * 
 * @return array|false - результат отправки или false при ошибке
 */
private function sendToBitrix24($companyId, $data, $debug = false)
```

### Маппинг полей

| Поле сайта | Поле Bitrix24 | Тип | Описание |
|------------|---------------|-----|----------|
| `OS_COMPANY_NAME` | `TITLE` | string | Название компании |
| `OS_COMPANY_INN` | `UF_CRM_INN` | string | ИНН компании |
| `OS_COMPANY_CITY` | `UF_CRM_1669208295583` | string | Город/Адрес |
| `OS_COMPANY_PHONE` | `PHONE[0][VALUE]` | array | Телефон (тип: WORK) |
| `OS_COMPANY_EMAIL` | `EMAIL[0][VALUE]` | array | Email (тип: WORK) |
| `OS_COMPANY_WEB_SITE` | `WEB[0][VALUE]` | array | Веб-сайт (тип: WORK) |

### Интеграция в процесс обновления

```php
// В методе updateCompanyProfile() после успешного обновления:

// Синхронизируем данные с Bitrix24
$b24SyncSuccess = false;
if (!empty($company['OS_COMPANY_B24_ID'])) {
    $b24Result = $this->sendToBitrix24($company['OS_COMPANY_B24_ID'], $data);
    $b24SyncSuccess = !empty($b24Result);
}

return [
    'success' => true,
    'message' => 'Данные компании успешно обновлены',
    'data' => [
        'company_id' => $companyId,
        'company_code' => $companyCode,
        'b24_synced' => $b24SyncSuccess  // <-- Статус синхронизации
    ]
];
```

## 🛡️ Обработка ошибок

### Принцип "fail-safe"

**Важно:** Ошибки синхронизации с Bitrix24 **НЕ прерывают** процесс обновления на сайте.

```php
try {
    $result = sendRequestB24('crm.company.update', [
        'id'     => $companyId,
        'fields' => $b24Fields,
    ], $debug);

    return $result;
} catch (\Exception $e) {
    // Логируем ошибку, но НЕ прерываем процесс
    error_log('Bitrix24 company update error: ' . $e->getMessage());
    return false;
}
```

### Сценарии ошибок

| Сценарий | Поведение системы |
|----------|------------------|
| Ошибка подключения к B24 | Данные на сайте обновлены, `b24_synced: false` |
| Неверный ID компании в B24 | Данные на сайте обновлены, ошибка залогирована |
| Таймаут запроса к B24 | Данные на сайте обновлены, `b24_synced: false` |
| Успешная синхронизация | Данные обновлены везде, `b24_synced: true` |

## 🔍 Отладка

### Включение режима отладки

Для включения детального логирования запросов к B24:

```php
// В методе updateCompanyProfile() измените:
$b24Result = $this->sendToBitrix24($company['OS_COMPANY_B24_ID'], $data, true);
//                                                                       ^^^^ debug mode
```

### Проверка логов

Ошибки логируются в стандартный error_log PHP:

```php
error_log('Bitrix24 company update error: ' . $e->getMessage());
```

Путь к логу зависит от конфигурации сервера (обычно `/var/log/php_errors.log`).

## ⚙️ Настройка полей B24

### Если поля отличаются

Кастомные поля в B24 могут иметь разные названия в зависимости от настройки CRM.

**Как узнать правильные названия полей:**

1. Выполните запрос к B24:
```php
$fields = sendRequestB24('crm.company.fields', []);
pre($fields);
```

2. Найдите нужное поле по описанию

3. Обновите маппинг в методе `sendToBitrix24()`:
```php
// Было:
$b24Fields['UF_CRM_1669208295583'] = $data['OS_COMPANY_CITY'];

// Стало (ваше значение):
$b24Fields['UF_CRM_YOUR_FIELD_NAME'] = $data['OS_COMPANY_CITY'];
```

## 📊 Мониторинг синхронизации

### Проверка статуса синхронизации

В ответе AJAX теперь доступно поле `b24_synced`:

```javascript
fetch('/ajax.php', {
    method: 'POST',
    body: formData
})
.then(response => response.json())
.then(data => {
    if (data.success) {
        console.log('Обновлено на сайте');
        
        if (data.b24_synced) {
            console.log('✓ Синхронизировано с B24');
        } else {
            console.warn('⚠ Не синхронизировано с B24');
        }
    }
});
```

### Статистика синхронизации

Для сбора статистики можно добавить логирование успешных синхронизаций:

```php
// В методе sendToBitrix24() после успешной отправки:
if ($result && !empty($result['result'])) {
    $logFile = $_SERVER['DOCUMENT_ROOT'] . '/log/b24_company_sync.log';
    file_put_contents($logFile, 
        date('Y-m-d H:i:s') . " | Company ID: $companyId | Success\n", 
        FILE_APPEND
    );
}
```

## 🔐 Безопасность

### Проверка прав доступа

Синхронизация происходит **только после** проверки прав через `checkEditPermission()`:

```php
// В ajax.php:
$permissionCheck = $company->checkEditPermission($companyId, $USER->GetID());
if (!$permissionCheck['has_access']) {
    // Отказано - синхронизация не произойдет
}
```

### Конфиденциальность данных

- Используется защищенное соединение с B24 (HTTPS)
- Токен доступа хранится в `init.php`
- Логи не содержат чувствительных данных
