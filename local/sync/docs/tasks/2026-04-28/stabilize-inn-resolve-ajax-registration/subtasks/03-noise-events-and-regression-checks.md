# Subtask 03: Noise events and regression checks

## Родительская задача
- [task.md](../task.md)

## Статус
- `todo`

## Цель
Снизить шум в логах и подтвердить регрессионно, что диагностические события отражают реальное поведение резолва ИНН в ajax-регистрации.

## Объем работ
- Удалить или заменить события, которые дублируют смысл или вводят в заблуждение.
- Оставить минимальный, но достаточный набор диагностических событий.
- Подготовить и пройти проверочный чеклист на ключевых сценариях.
- Сформировать краткую памятку интерпретации логов для поддержки.

## Входы
- Обновленная логика резолва из `02-resolve-pipeline-fix`.
- Целевой контракт логов из `01-baseline-and-logging-contract`.

## Выходы
- Стабилизированный поток событий в логе без ложных сигналов дублей.
- Отчет по проверкам (какие кейсы пройдены и что наблюдается в логе).

## Зависимости
- Готовность подзадачи 02.
- Доступ к среде/данным для прогонки сценариев регистрации.

## Риски
- Недостаточная детализация логов после очистки может усложнить future-debug.
- Неполное покрытие кейсов оставит скрытый шум в редких ветках.

## Индивидуальные критерии готовности
- В логах отсутствуют двусмысленные события, которые трактуются как дубль без exact-match.
- Для каждого ключевого сценария есть ожидаемый и фактический набор событий.
- Проверочный чеклист пройден без критических расхождений.

## Чеклист проверки результата
- [ ] Пройден кейс: ложные property-кандидаты без exact-match.
- [ ] Пройден кейс: валидный exact-match (дубль подтверждается корректно).
- [ ] Пройден кейс: отсутствие кандидатов (чистый `no_match`).
- [ ] Подготовлена краткая памятка интерпретации логов.
- [ ] Обновлен статус подзадачи и синхронизирован с `task.md`.
