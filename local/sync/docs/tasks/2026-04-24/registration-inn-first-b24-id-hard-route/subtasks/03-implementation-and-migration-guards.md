# Subtask 03: Реализация и миграционные оговорки

## Родительская задача
- `../task.md`

## Цель
Подготовить исполнимый технический план внедрения новой логики регистрации с учетом переходного периода и рисков исторических данных.

## Объем работ
- Определить последовательность внедрения по компонентам (ajax/usersync/rest/integration points).
- Выделить минимальный безопасный инкремент и стратегию поэтапного включения.
- Описать миграционные оговорки для записей без `OS_COMPANY_B24_ID` и с невалидной связью.
- Согласовать критерии rollback и действия при частичной деградации.

## Входы
- Согласованный to-be контракт из Subtask 02.
- Аудит текущей цепочки из Subtask 01.
- Ограничения релизного окна и эксплуатационные требования.

## Выходы
- Поэтапный план реализации для `go-senior-developer-executor`.
- Миграционная стратегия и guardrails переходного периода.
- Список обязательных технических проверок перед выкладкой.

## Зависимости
- Одобрение архитектурного контракта `go-teamlead`.
- Поддержка тестовой среды для dry-run сценариев.

## Риски
- Неполные данные linkage при старых регистрациях.
- Перекрестное влияние на связанные процедуры создания/обновления контактов.
- Ошибки при конкурентных регистрациях одного ИНН.

## Индивидуальные критерии готовности
- Есть пошаговый план внедрения с четкими входами/выходами этапов.
- Миграционные правила покрывают случаи "нет связи", "битая связь", "дубли в B24".
- Определены rollback-условия и критерии остановки релиза.
- План подтвержден `go-teamlead` и передан исполнителю.
