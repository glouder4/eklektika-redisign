# Компонент редактирования профиля пользователя

## Обзор

Компонент `online-service:user.profile.edit` добавляет функционал редактирования профилей пользователей и управления их привязками к компаниям.

## Дата создания
Октябрь 2025

## Местоположение

### Компонент
- **Путь**: `local/components/online-service/user.profile.edit/`
- **Класс**: `OnlineServiceUserProfileEditComponent`

### Страницы
- **Редактирование**: `/company/user/{USER_ID}/edit/`
- **Файл**: `company/user/edit/index.php`

### URL Маршруты
Настроены в `urlrewrite.php`:
- `/company/user/{USER_ID}/edit/?type=profile` - редактирование профиля
- `/company/user/{USER_ID}/edit/?type=companies` - управление компаниями

## Функционал

### 1. Редактирование профиля (type=profile)

#### Поля редактирования:
- **Личная информация**:
  - Имя (обязательное)
  - Фамилия (обязательное)
  
- **Рабочая информация**:
  - Должность
  
- **Контакты**:
  - Мобильный телефон (с маской ввода)
  - Рабочий телефон (с маской ввода)
  - Личный телефон (с маской ввода)
  - Email (обязательное, недоступно для редактирования руководителем)

- **Фотография**:
  - Загрузка нового фото (макс. 5MB)
  - Удаление текущего фото
  - Предпросмотр в реальном времени

#### Особенности:
- AJAX отправка без перезагрузки страницы
- Валидация на клиенте и сервере
- Автоматическое форматирование телефонов
- Уведомления об успехе/ошибке
- Редирект на профиль после сохранения

### 2. Управление компаниями (type=companies)

#### Функции:
- **Просмотр компаний**:
  - Таблица со списком компаний
  - Отображение роли (Руководитель/Сотрудник)
  - Статус компании (Активно/На модерации)
  - Бейдж головной компании холдинга

- **Действия с компаниями**:
  - Отвязка пользователя от компании
  - Смена роли сотрудник ↔ руководитель (только для админа)
  - Привязка к новой компании

- **Модальные окна**:
  - Подтверждение перед отвязкой
  - Подтверждение перед сменой роли

## Права доступа

### Матрица прав:

| Действие | Админ | Сам пользователь | Руководитель |
|----------|-------|------------------|--------------|
| Редактировать ФИО | ✅ | ✅ | ✅ |
| Редактировать должность | ✅ | ✅ | ✅ |
| Редактировать телефоны | ✅ | ✅ | ✅ |
| Редактировать email | ✅ | ✅ | ❌ |
| Редактировать фото | ✅ | ✅ | ✅ |
| Привязать к компании | ✅ | ❌ | ✅* |
| Отвязать от компании | ✅ | ❌ | ✅* |
| Сменить роль в компании | ✅ | ❌ | ❌ |

_* Только для своих компаний_

### Логика определения прав:

```php
// Уровни доступа
'admin'  // Администратор - полный доступ
'self'   // Сам пользователь - свой профиль
'boss'   // Руководитель - сотрудники своих компаний
'none'   // Нет доступа
```

## Используемые свойства инфоблока компаний

- `OS_COMPANY_BOSS` (ID: множественное) - Руководители компании
- `OS_COMPANY_USERS` (ID: множественное) - Сотрудники компании
- `OS_IS_MARKETING_AGENT` (список) - Статус рекламного агента (YES/NO)
- `OS_COMPANY_IS_HEAD_OF_HOLDING` (Да/Нет) - Головная компания холдинга
- `OS_HOLDING_OF` (ID: элемент) - Принадлежность к холдингу

## Используемые пользовательские поля

### CUser поля:
- `NAME` - Имя
- `LAST_NAME` - Фамилия
- `WORK_POSITION` - Должность
- `PERSONAL_MOBILE` - Мобильный телефон
- `WORK_PHONE` - Рабочий телефон
- `PERSONAL_PHONE` - Личный телефон
- `EMAIL` - Email
- `PERSONAL_PHOTO` - Фотография

## AJAX API

### Endpoint: `/local/components/online-service/user.profile.edit/ajax.php`

#### Формат запроса:
```json
{
  "action": "saveProfile",
  "data": {
    "userId": 123,
    "fields": {
      "NAME": "Иван",
      "LAST_NAME": "Иванов"
    }
  },
  "sessid": "session_id"
}
```

#### Формат ответа:
```json
{
  "data": {
    "success": true,
    "message": "Профиль успешно обновлен"
  }
}
```

### Доступные действия:

1. **saveProfile** - Сохранение профиля
2. **attachToCompany** - Привязка к компании
3. **detachFromCompany** - Отвязка от компании
4. **changeRole** - Смена роли

## Безопасность

### Реализованные меры:

1. **Аутентификация**:
   - Проверка авторизации пользователя
   - Валидация sessid (CSRF защита)

2. **Авторизация**:
   - Проверка прав перед каждым действием
   - Разграничение доступа по уровням

3. **Валидация данных**:
   - Проверка типов данных
   - Санитизация входных значений
   - Проверка размера и типа файлов

4. **Защита email**:
   - Руководители не могут менять email сотрудников
   - Предотвращение перехвата аккаунтов
