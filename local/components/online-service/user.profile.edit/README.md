# Компонент редактирования профиля пользователя

## Описание

Компонент `online-service:user.profile.edit` предназначен для редактирования профиля пользователя и управления его привязками к компаниям.

## Возможности

### 1. Редактирование профиля (`type=profile`)

- Изменение ФИО пользователя
- Изменение должности
- Редактирование контактных данных (телефоны, email)
- Загрузка/удаление фотографии

### 2. Управление компаниями (`type=companies`)

- Просмотр списка компаний пользователя с указанием роли (Руководитель/Сотрудник)
- Привязка пользователя к компаниям
- Отвязка пользователя от компаний
- Изменение роли в компании (доступно только администратору)

## Параметры компонента

| Параметр | Тип | Описание | По умолчанию |
|----------|-----|----------|--------------|
| USER_ID | int | ID редактируемого пользователя | - |
| TYPE | string | Тип редактирования: `profile` или `companies` | `profile` |
| CACHE_TIME | int | Время кеширования (секунды) | 0 |

## Права доступа

### Уровни доступа:

1. **admin** - Администратор
   - Полный доступ ко всем функциям
   - Может редактировать любые поля любого пользователя
   - Может управлять ролями в компаниях

2. **self** - Сам пользователь
   - Может редактировать свой профиль
   - Может редактировать все поля включая email
   - Не может сам управлять своими компаниями

3. **boss** - Руководитель компании
   - Может редактировать профили своих сотрудников
   - Не может изменять email сотрудников (безопасность)
   - Может управлять сотрудниками в своих компаниях

## Использование

### Базовый вызов

```php
<?$APPLICATION->IncludeComponent(
    "online-service:user.profile.edit",
    ".default",
    Array(
        "USER_ID" => $userId,
        "TYPE" => "profile",
        "CACHE_TIME" => 0,
    )
);?>
```

### URL маршруты

- `/company/user/{USER_ID}/edit/?type=profile` - Редактирование профиля
- `/company/user/{USER_ID}/edit/?type=companies` - Управление компаниями

## AJAX методы

Компонент поддерживает следующие AJAX действия через `ajax.php`:

### saveProfile
Сохранение данных профиля

**Параметры:**
- `userId` (int) - ID пользователя
- `fields` (array) - Массив полей для обновления

**Возврат:**
```json
{
  "success": true,
  "message": "Профиль успешно обновлен"
}
```

### attachToCompany
Привязка пользователя к компании

**Параметры:**
- `userId` (int) - ID пользователя
- `companyId` (int) - ID компании
- `role` (string) - Роль: `employee` или `boss`

**Возврат:**
```json
{
  "success": true,
  "message": "Пользователь успешно привязан к компании"
}
```

### detachFromCompany
Отвязка пользователя от компании

**Параметры:**
- `userId` (int) - ID пользователя
- `companyId` (int) - ID компании

**Возврат:**
```json
{
  "success": true,
  "message": "Пользователь успешно отвязан от компании"
}
```

### changeRole
Изменение роли пользователя в компании

**Параметры:**
- `userId` (int) - ID пользователя
- `companyId` (int) - ID компании
- `newRole` (string) - Новая роль: `employee` или `boss`

**Возврат:**
```json
{
  "success": true,
  "message": "Роль успешно изменена"
}
```

## JavaScript API

Компонент создает экземпляр класса `UserProfileEdit` для управления формами:

```javascript
new UserProfileEdit({
    componentPath: '/path/to/component',
    userId: 123,
    type: 'profile',
    sessid: 'session_id'
});
```

### Методы класса:

- `initProfileForm()` - Инициализация формы профиля
- `initCompaniesManagement()` - Инициализация управления компаниями
- `handleProfileSubmit(e)` - Обработка отправки формы профиля
- `handlePhotoUpload(e)` - Обработка загрузки фото
- `handlePhotoDelete()` - Обработка удаления фото
- `handleAttach()` - Привязка к компании
- `handleDetach(e)` - Отвязка от компании
- `handleChangeRole(e)` - Смена роли
- `showNotification(message, type)` - Показ уведомлений
- `showConfirmModal(message, onConfirm)` - Показ модального окна подтверждения

## Стили

Компонент использует следующие основные CSS классы:

- `.user-profile-edit-wrapper` - Главный контейнер
- `.profile-edit-form` - Форма редактирования
- `.form-section` - Секция формы
- `.companies-table` - Таблица компаний
- `.notification` - Уведомление
- `.modal` - Модальное окно

## Интеграция с системой

Компонент интегрирован с:

- **Системой пользователей** (`CUser`) - для обновления профилей
- **Инфоблоками** (ID: 57) - для управления компаниями
- **Системой прав** - проверка доступа на основе ролей

## Безопасность

- Проверка CSRF токена (`bitrix_sessid()`)
- Валидация прав доступа перед каждым действием
- Санитизация входных данных
- Ограничение доступа к изменению email для руководителей
- Проверка типа и размера загружаемых файлов

## Особенности

1. **Маски телефонов** - автоматическое форматирование при вводе
2. **Предпросмотр фото** - мгновенный предпросмотр при загрузке
3. **AJAX обработка** - все действия без перезагрузки страницы
4. **Уведомления** - визуальная обратная связь для пользователя
5. **Модальные окна** - подтверждение критических действий
6. **Адаптивность** - работает на мобильных устройствах

## Зависимости

- Bitrix Framework (main, iblock модули)
- Инфоблок компаний (ID: 57)
- Свойства компаний:
  - `OS_COMPANY_BOSS` - руководители
  - `OS_COMPANY_USERS` - сотрудники
  - `OS_IS_MARKETING_AGENT` - статус рекламного агента
  - `OS_COMPANY_IS_HEAD_OF_HOLDING` - головная компания холдинга
